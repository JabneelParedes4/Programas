Este módulo utiliza información temporal proveniente del archivo dataset_walmart.csv, el cual contiene registros históricos de transacciones comerciales realizadas entre enero y marzo de 2026.

Para el análisis de horarios de concurrencia se seleccionaron variables relacionadas con la fecha, hora y frecuencia de compras, debido a que permiten identificar patrones de comportamiento comercial y tendencias de asistencia dentro de las sucursales.

Las variables utilizadas fueron:

  • Fecha: permite identificar comportamiento diario, semanal y mensual.
  • Hora: utilizada para detectar horarios pico y niveles de concurrencia.
  • Invoice ID: empleado para contabilizar clientes únicos y tickets registrados.
  • Sucursal: permite comparar tendencias entre diferentes puntos de venta.

Los datos son de naturaleza temporal y secuencial, ya que representan eventos comerciales ocurridos en diferentes momentos del tiempo. El sistema procesa aproximadamente 79,875 registros históricos y cerca de 18,294 tickets únicos para modelar tendencias, horarios pico, comportamiento por quincena y diferencias de concurrencia entre días festivos y días normales.
